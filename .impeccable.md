# Cmux Mobile Design Notes

## Design Context

### Users
Power users who monitor and control Claude Code, Codex, and cmux sessions from iOS/Android while away from the laptop. They need fast read/act loops: inspect terminal output, edit or resend commands, approve actions, and attach files without fighting mobile ergonomics.

### Brand Personality
Industrial, calm, precise. The interface should feel like a compact field console for serious work: dense enough for terminal output, readable under phone constraints, and confident without decorative AI-style gloss.

### Aesthetic Direction
High-contrast mobile utility terminal. Dark theme is primary, with warm restrained accent and text tuned for legibility. It should feel closer to a sharp cmux companion than a generic chat app. Avoid neon cyberpunk, glassmorphism, gradient text, card spam, and left accent stripes.

### Design Principles
- Prioritize terminal readability over decorative styling: stable monospace, tight but not cramped line-height, preserved spacing, and user-controlled scale.
- Keep mobile controls thumb-reachable and explicit: small surface area, clear labels, no hidden critical actions.
- Preserve real terminal behavior unless a mobile convenience has a clear guardrail.
- Prefer local, private workflows: attachments become local bridge files and insert paths, not cloud uploads.
- Refresh PWA updates in-place with a visible update action instead of forcing reinstall.
